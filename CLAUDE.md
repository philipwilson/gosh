# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

gosh is an educational Unix shell implemented in Go. It follows bash semantics and uses explicit system calls (no libraries outside the Go standard library). Clarity over cleverness.

## Commands

```bash
go build -ldflags "-X main.version=$(git describe --tags --always --dirty)" -o gosh .  # build with version
go build -o gosh .        # build (version defaults to "dev")
go test ./...              # run all tests
go test ./lexer/ -v        # run one package's tests
go test ./lexer/ -run TestSingleQuotes  # run a single test
```

Test interactively: `printf 'echo hello | tr a-z A-Z\nexit\n' | ./gosh`

## Architecture

Shell input flows through phases with clean boundaries:

```
Input → Lexer → []Token → Heredoc Resolution → Parser → AST → Executor (expands each entry lazily before execution)
```

- **Lexer** (`lexer/`): Converts raw input to tokens. Handles single quotes (literal), double quotes (with `\` escapes for `" \ $ \``), backslash escapes, comments (unquoted `#` skips rest of line), and operator recognition (`|`, `<`, `>`, `>>`, `>&N`, `<&N`, `<<`, `<<-`, `;`, `;;`, `&&`, `||`, `&`, `(`, `)`). Newlines produce `TOKEN_SEMI` (suppressed after operators, `;;`, `(`, `<<`/`<<-`, and other separators; `)` no longer suppresses newlines to support multi-line array assignments) to support multi-line input. A single digit before `>`, `>>`, `<`, or `<<` is absorbed as the fd number (e.g., `2>file`). Each WORD token carries `Parts []WordPart` preserving quoting context (`Unquoted`, `SingleQuoted`, `DoubleQuoted`, `CmdSubst`, `CmdSubstDQ`, `ProcSubstIn`, `ProcSubstOut`) for the expander. Process substitution (`<(cmd)`, `>(cmd)`) is recognized when `<` or `>` is immediately followed by `(` (checked before `<<`/`>>`); the inner command is read via `readCmdSubst` and produces a `TOKEN_WORD` with a single `ProcSubstIn` or `ProcSubstOut` part. `$(cmd)` supports nesting and respects internal quoting; `` `cmd` `` supports `\` escapes. Redirect tokens carry an `Fd` field (-1 = use default). Here documents (`<<DELIM`, `<<-DELIM`) produce `TOKEN_HEREDOC` with a `HeredocInfo` struct carrying the delimiter, tab-strip flag, and expansion flag. `<<'DELIM'` and `<<"DELIM"` suppress expansion. `ResolveHeredocs` fills in the body by reading subsequent lines until the delimiter is found. `LexHeredocBody` re-lexes expandable bodies as double-quoted content to produce proper `CmdSubstDQ`/`ArithSubstDQ` parts. Here strings (`<<<word`) produce `TOKEN_HERESTRING`; the parser reads the next word and creates a `REDIR_HERESTRING` redirect. Arithmetic commands (`(( expr ))`) are recognized when `(` is followed by `(`; the expression is read until `))` and produces `TOKEN_ARITH_CMD`.

- **Parser** (`parser/`): Recursive descent parser producing an AST. Grammar: `list → pipeline ((; | && | || | &) pipeline)*`, `pipeline → command (| command)*`, `command → compound_cmd | simple_cmd`, `compound_cmd → if_cmd | while_cmd | until_cmd | for_cmd | select_cmd | case_cmd | func_def | subshell | dbl_bracket`, `if_cmd → 'if' list 'then' list ('elif' list 'then' list)* ('else' list)? 'fi'`, `while_cmd → 'while' list 'do' list 'done'`, `until_cmd → 'until' list 'do' list 'done'`, `for_cmd → 'for' NAME 'in' word... ';' 'do' list 'done'`, `case_cmd → 'case' word 'in' (pattern ('|' pattern)* ')' list ';;')* 'esac'`, `func_def → NAME '(' ')' '{' list '}'`, `select_cmd → 'select' NAME 'in' word... ';' 'do' list 'done'`, `arith_for → 'for' '((' init ';' cond ';' step '))' 'do' list 'done'`. AST nodes: `List` (sequence with operators), `Pipeline` (pipe-connected commands), `Command` interface (implemented by `SimpleCmd`, `IfCmd`, `WhileCmd`, `UntilCmd`, `ForCmd`, `ArithForCmd`, `SelectCmd`, `CaseCmd`, `FuncDef`, `ArithCmd`, `SubshellCmd`, `DblBracketCmd`), `SimpleCmd` (assignments + args as `[]lexer.Word` + redirections), `IfCmd` (clauses with condition/body lists + optional else), `WhileCmd` (condition + body), `UntilCmd` (condition + body, inverted check), `ForCmd` (variable name + word list + body), `CaseCmd` (word + clauses with patterns and body). `CloneList` deep-copies AST subtrees so loops can re-expand on each iteration. `FuncDef` (function name + body list). Reserved words (`then`, `elif`, `else`, `fi`, `do`, `done`, `in`, `esac`, `}`, `select`) terminate list parsing via stop words. `TOKEN_DSEMI` (`;;`) also terminates list parsing for case clause bodies. Recognizes `NAME=VALUE`, `NAME=(word...)` (array), `NAME+=(word...)` (array append), and `NAME[EXPR]=VALUE` (indexed) assignments before command words. `Assignment` struct has `Name`, `Value`, `Index`, `Array`, `Append` fields. `&` marks the preceding pipeline for background execution.

- **Expander** (`expander/`): Seven-phase expansion on the AST: (0) brace expansion (`{a,b,c}`, `{1..5}`, `{a..e}`) on args only — comma lists, integer/letter sequences with optional zero-padding, nesting, cartesian products; only unquoted braces/commas participate, `${...}` skipped, single-element `{a}` and empty `{}` stay literal (`expander/brace.go`); (1) tilde expansion (`~` → `$HOME`, `~user` → user's home dir, only unquoted); (2a) arithmetic expansion (`$((expr))`) via recursive descent evaluator (`expander/arith.go`) — supports `+`, `-`, `*`, `/`, `%`, comparisons, logical/bitwise operators, ternary `?:`, parenthesized grouping, variable references (bare `x` or `$x`), assignment operators (`=`, `+=`, `-=`, `*=`, `/=`, `%=`), and pre/post increment/decrement (`++x`, `x++`, `--x`, `x--`). Assignments write back to shell variables via a `SetFunc` callback; (2b) command substitution (`$(cmd)` and `` `cmd` ``) via a `SubstFunc` callback that recursively lex→parse→expand→executes, capturing stdout through a pipe — unquoted results marked `Expanded` (subject to splitting/globs), double-quoted results stay `DoubleQuoted`; (3) variable expansion (`$VAR`, `${VAR}`, `$?`, `$$`, `$!`, `$#`, `$@`, `$*`, `$0`-`$9`, `$RANDOM`, `$SECONDS`) respecting quoting — no expansion in SingleQuoted parts, unquoted results use `expandDollarParts` to produce structured `Expanded` parts preserving literal/expansion boundaries; parameter expansion (`${var:-word}`, `${var:+word}`, `${var:=word}`, `${var:?msg}`, `${#var}`, `${var#pat}`, `${var##pat}`, `${var%pat}`, `${var%%pat}`, `${var/pat/rep}`, `${var//pat/rep}`, `${var:offset}`, `${var:offset:length}`) — colon variants test for empty, non-colon variants test for unset only, `#`/`##` remove shortest/longest prefix, `%`/`%%` remove shortest/longest suffix, `/` replaces first match, `//` replaces all matches (longest match at each position), `:offset` and `:offset:length` extract substrings (negative offset = from end, negative length = stop before end); case conversion (`${var^}` uppercase first, `${var^^}` uppercase all, `${var,}` lowercase first, `${var,,}` lowercase all, with optional pattern `${var^^[pattern]}`); array subscript expansion (`${arr[N]}`, `${arr[@]}`, `${arr[*]}`, `${#arr[@]}`), patterns use `filepath.Match` with a wrapper (`patternMatch`) that replaces `/` with `\x00` so `*` matches path separators; `"${arr[@]}"` produces one word per array element (same for `"$@"`), `"${arr[*]}"` stays space-joined; `LookupArrayFunc` callback enables this by returning individual elements for `@` subscripts; `expandVarsInWordMulti` handles splitting a single input word into multiple output words; (3.5) word splitting (IFS field splitting) on args only — splits `Expanded` parts on IFS characters (default `" \t\n"`), following POSIX rules: IFS whitespace is trimmed/collapsed, non-whitespace IFS chars each act as individual delimiters. Empty unquoted expansions are removed, quoted empty strings preserved. One word can become many; (4) glob expansion (`*`, `?`, `[...]`) on unquoted args only, using `filepath.Glob`. Builds glob patterns that escape metacharacters in quoted parts.

- **Executor** (`exec.go`): Walks the AST with stdin/stdout threaded through the entire call chain. All execution functions — `execList`, `execPipeline`, `execCommand`, `execSimple`, `execIf`, `execWhile`, `execFor`, `execCase`, `execFunction` — accept `stdin, stdout *os.File` parameters. `runLine` passes `os.Stdin`/`os.Stdout`; `cmdSubst` passes `os.Stdin`/pipe-writer, so command substitution output is captured without swapping globals. `execList` performs **lazy per-entry expansion**: each pipeline is expanded just before execution, so assignments in earlier entries are visible to later ones (e.g., `X=1; echo $X` works). Dispatches `Command` interface via type switch (`execCommand`) to `execSimple`, `execIf`, `execWhile`, `execUntil`, `execFor`, `execSelect`, `execCase`, `execArithCmd`, `execArithFor`, `execSubshell`, `execDblBracket`, or stores `FuncDef` in the function table. `execSelect` displays a numbered menu on stderr, reads user selection from stdin, sets the loop variable and `REPLY`, and runs the body; empty input redisplays the menu, EOF exits, `break`/`continue` work normally. PS3 variable controls the prompt (default `#? `). `execUntil` is identical to `execWhile` but with inverted condition (runs body while condition is non-zero). `execArithCmd` expands variables in the expression, evaluates via `EvalArith`, and returns 0 if non-zero, 1 if zero (bash semantics). `execSubshell` saves shell state (vars, arrays, exports, funcs, aliases, positional params), runs the body, then restores state — variable changes inside the subshell do not affect the parent. `execDblBracket` (`dblbracket.go`) expands variables without word splitting or glob expansion, then evaluates via recursive descent supporting `&&`, `||`, `!`, `()`, string tests (`-z`, `-n`), file tests (`-e`, `-f`, `-d`, `-r`, `-w`, `-x`, `-s`), string comparison (`<`, `>`), integer comparison (`-eq`, `-ne`, `-lt`, `-le`, `-gt`, `-ge`), and pattern matching (`==`, `!=` — glob patterns when RHS unquoted, literal when quoted). `execIf` clones and expands each condition/body lazily — only the taken branch is expanded. `execWhile` clones and re-expands the condition and body on each iteration via `CloneList`. `execFor` expands the word list once (variables + globs), then iterates with the loop variable set, cloning the body for each iteration. `execArithFor` evaluates init/cond/step expressions via `EvalArith`; empty condition means infinite loop. `execCase` expands the subject word fully, then matches each clause's patterns (variable-expanded only, no glob file expansion) using `filepath.Match`; the first matching clause's body is expanded and executed. Spawns processes via `os.StartProcess` with `SysProcAttr{Setpgid: true}` for process group isolation. Wires pipes with `os.Pipe()`. Applies redirections (`<`, `>`, `>>`, `2>`, `2>&1`, etc.) using a `[3]*os.File` fd table (stdin/stdout/stderr). Manages terminal foreground group via `tcsetpgrp` (TIOCSPGRP ioctl), gated by `substDepth == 0` to avoid terminal control inside command substitutions. User-defined functions are looked up before builtins; `execFunction` saves/restores positional parameters (`$1`-`$9`, `$#`, `$@`), clones the body, and clears `returnFlag` on exit. Function redirects are honored (e.g., `f > file`). Runs builtins in-process for standalone commands; in pipelines they fall through to external lookup. Implements `&&`/`||` short-circuit evaluation. Per-command assignments are temporary for builtins (save/restore). Uses `syscall.Wait4` with `WUNTRACED` to detect stopped processes (Ctrl-Z). Background pipelines (`&`) run without terminal foreground and are tracked in the job table; the leader PID is stored in `state.lastBgPid` (`$!`). `exec` is intercepted in `execSimple` before builtin dispatch: `exec cmd args...` replaces the shell process via `syscall.Exec` (fires EXIT trap first), `exec` with redirects only permanently redirects shell fds via `syscall.Dup2`, `exec` alone is a no-op. Process substitution (`<(cmd)`, `>(cmd)`) is handled by `processProcSubsts` in `execSimple`: for each `ProcSubstIn`/`ProcSubstOut` arg, creates a FIFO via `syscall.Mkfifo`, spawns a goroutine with a `cloneShellState` to run the inner command reading from/writing to the FIFO, and replaces the arg with the FIFO path. Cleanup removes FIFOs after the command finishes.

## Key Design Details

- `os.StartProcess` / `syscall` for process management, not `os/exec.Cmd` — the plumbing should be visible. Source split: `main.go` (state/REPL), `exec.go` (execution), `builtins.go` (builtins), `dblbracket.go` (`[[ ]]` evaluator), `test_builtin.go` (`test`/`[` evaluator), `jobs.go` (job table), `terminal.go` (ioctl wrappers), `complete.go` (tab completion)
- `exec.LookPath` is used for PATH resolution
- Phases are separate packages with explicit data passed between them (tokens, AST nodes with `lexer.Word` parts)
- Redirections override pipe defaults (e.g., `sort < file | head` uses file as sort's stdin). Supports fd-specific redirects (`2>file`, `2>>file`), fd duplication (`2>&1`, `>&2`), here documents (`<<DELIM`, `<<-DELIM`, `<<'DELIM'`), and here strings (`<<<word`). Heredoc/herestring redirects create a pipe, write the body (+ trailing newline for herestrings), and pass the read end as stdin.
- Exit status: last pipeline command determines status; signal kills → 128+signum
- Process groups: each pipeline gets its own group; SIGTTOU uses `signal.Ignore` (SIG_IGN required for `tcsetpgrp` from background); SIGINT/SIGTSTP use `signal.Notify` (caught handler resets to SIG_DFL across exec, so children respond to Ctrl-C/Ctrl-Z); SIGCHLD left at SIG_DFL
- Terminal control only when interactive (`isatty` via TIOCGPGRP probe) and `substDepth == 0` (not inside command substitution)
- Job control: stopped (Ctrl-Z) and background (`&`) jobs tracked in a job table (`jobs.go`). `fg` resumes in foreground with `SIGCONT` + `tcsetpgrp`, `bg` resumes in background. `wait` blocks until background jobs finish (`wait` = all, `wait %N` = job N, `wait PID` = specific PID); returns exit status of waited process. `$!` holds PID of last background command (set in `execBackground`). Finished background jobs are reaped before each prompt via `Wait4(-1, WNOHANG)`.
- `exec` builtin: intercepted as a special case in `execSimple` (before builtin dispatch) because it needs access to the `SimpleCmd` redirect list. `exec cmd args...` replaces the shell via `syscall.Exec` (saves history, fires EXIT trap). `exec > file` permanently redirects shell fds via `syscall.Dup2` + updates `os.Stdin`/`os.Stdout`/`os.Stderr`. `exec` alone is a no-op. Stub entry in `builtins` map for tab completion and `type`/`command -v`.
- Process substitution: `<(cmd)` and `>(cmd)` are recognized by the lexer as `TOKEN_WORD` with `ProcSubstIn`/`ProcSubstOut` parts. The executor's `processProcSubsts` creates FIFOs (`syscall.Mkfifo`), spawns goroutines with `cloneShellState` to run inner commands, and replaces args with FIFO paths. `<(cmd)` goroutine writes cmd stdout to FIFO; `>(cmd)` goroutine reads FIFO as cmd stdin. Cleanup waits for goroutines and removes temp FIFOs.
- Functions: `fname() { body; }` defines a function stored in `shellState.funcs`. Called like commands with positional parameters (`$1`-`$9`, `$#`, `$@`/`$*`). `return [N]` exits the function with status N. Supports recursion — positional params are saved/restored per call. `local var[=value]` declares function-scoped variables (dynamic scoping, matching bash); `localScopes` stack on `shellState` tracks saved values, restored on function exit. Function redirects work (e.g., `f > file`). Function names are included in tab completion.
- Builtins (cd, pwd, echo, printf, exit, export, unset, true, false, break, continue, return, shift, local, alias, unalias, test, [, read, eval, source/., history, jobs, fg, bg, wait, exec, trap, let) run in-process with redirect support; builtins receive both stdin and stdout. cd supports `-` (OLDPWD) and updates PWD/OLDPWD
- Traps: `trap 'command' SIGNAL` sets a handler, `trap - SIGNAL` removes, `trap '' SIGNAL` ignores, `trap` lists. Supports real signals (INT, TERM, HUP, QUIT, USR1, USR2) and pseudo-signals (EXIT, ERR, RETURN). Signal traps fire via `runPendingTraps` after each pipeline in `execList`. ERR fires on non-zero exit status. RETURN fires at the end of `execFunction`. EXIT fires before shell exit. `trapRunning` flag prevents recursive trap execution. Traps are saved/restored in subshells. `shellState` holds `traps map[string]string`, `pendingSignals map[string]bool`, `trapRunning bool`, `sigCh chan os.Signal`.
- Aliases: `alias name=value` defines aliases, `alias` lists them, `unalias name` / `unalias -a` removes them. Aliases are expanded in `expandAliases` (in `main.go`) before parsing — command-position words matching an alias are replaced by re-lexing the alias value. If an alias value ends with a space, the next word is also checked. Circular aliases are detected via a `seen` set. Alias names are included in tab completion. Aliases take effect on the next input line (same as bash).
- `read` builtin (`builtins.go`): reads one line from stdin, splits by IFS into named variables (last gets remainder). No args → stores in `REPLY`. `-r` flag disables backslash escape processing. Returns 1 on EOF. `splitByIFS` handles POSIX IFS splitting rules.
- `test`/`[` builtin (`test_builtin.go`): recursive descent evaluator supporting string tests (`-z`, `-n`, `=`, `!=`), integer comparisons (`-eq`, `-ne`, `-lt`, `-le`, `-gt`, `-ge`), file tests (`-e`, `-f`, `-d`, `-r`, `-w`, `-x`, `-s`), logical operators (`!`, `-a`, `-o`), and parenthesized grouping. `[` requires closing `]`.
- Startup: interactive shells source `~/.goshrc` at startup (after editor init, before the REPL). Non-interactive shells and scripts do not source it, matching bash behavior.
- Script execution: `gosh script.sh` runs a file. `gosh -c 'command string'` executes a command string and exits (not interactive — no `.goshrc`, no editor, no prompt). Arguments after the command string: first becomes `$0`, rest become positional params (`$1`, `$2`, ...). `source`/`.` builtins execute a file in the current shell context (variables persist). Registered via `init()` to break the initialization cycle with the `builtins` map.
- Loop control: `break` and `continue` builtins set flags on `shellState`; `execList`, `execWhile`, `execUntil`, and `execFor` check `breakFlag`/`continueFlag` to exit or skip iterations. `loopDepth` tracks nesting to reject break/continue outside loops. `returnFlag` is also checked in loops and `execList` to propagate function returns through nested constructs.
- Multi-line input: `needsMore()` detects incomplete input (trailing `\`, unclosed quotes, trailing operators, parse errors ending with "got EOF"). Both interactive and non-interactive modes collect continuation lines. Trailing backslash joins lines directly; otherwise lines are joined with `\n`.
- Prompt formatting: `PS1` and `PS2` variables control the primary and continuation prompts. `formatPrompt` expands bash-style backslash escapes: `\u` (username), `\h` (short hostname), `\H` (full hostname), `\w` (cwd with ~ for home), `\W` (basename of cwd), `\$` (# if root, $ otherwise), `\t` (24h time HH:MM:SS), `\e` (ESC for ANSI colors), `\n` (newline), `\\` (literal backslash), `\[`/`\]` (ignored — editor uses relative cursor positioning). Default PS1: `\u@\h:\w\$ `, default PS2: `> `. Inherits PS1/PS2 from environment if already set.
- Debug builtins (`debug-tokens`, `debug-ast`, `debug-expanded`) toggle printing of tokens, pre-expansion AST, and post-expansion AST to stderr
- Indexed arrays: `shellState.arrays map[string][]string` stores indexed arrays separately from scalars. `arr=(a b c)` creates, `arr+=(d)` appends, `arr[N]=val` sets element, `${arr[N]}` reads, `${arr[@]}`/`${arr[*]}` expands all (skipping empty), `${#arr[@]}` counts non-empty elements, `unset 'arr[N]'` clears element. Subscripts are arithmetic-evaluated (support `$var`, `expr`). Bare `$arr` returns element 0. Arrays are not exported to children. `local -a` declares function-local arrays. `read -a name` splits line into array. `parseArrayRef` helper parses `name[subscript]` strings. `evalArraySubscript` in expander evaluates arithmetic subscripts. Arithmetic expressions (`$((arr[i]))`, `(( arr[i] = val ))`) support array element access and assignment.
- `shellState` holds variables, arrays, export set, function table, positional parameters, last exit status, `lastBgPid` (`$!`), `substDepth` (command substitution nesting), `startTime` (for `$SECONDS`), editor, terminal info, and job table
- Dynamic special variables: `$RANDOM` returns a random integer 0-32767 (via `math/rand`). `$SECONDS` returns elapsed seconds since shell start; assigning to `SECONDS` resets the timer (adjusts `startTime`).
- `BASH_REMATCH` array: set by `[[ string =~ regex ]]`. Element 0 is the full match, elements 1+ are capture groups. Cleared on no match. Uses Go `regexp.FindStringSubmatch`. Note: regex patterns with `(` in the RHS must be stored in a variable (e.g., `RE='([a-z]+)'; [[ x =~ $RE ]]`) because the lexer treats bare `(` as `TOKEN_LPAREN`.

## Line Editor

The `editor/` package provides interactive line editing and command history:

- **Terminal control** (`terminal_darwin.go`, `terminal_linux.go`): Platform-specific `tcgetattr`/`tcsetattr` via ioctl. Raw mode disables ICANON, ECHO, ISIG, IEXTEN, ICRNL, OPOST; sets VMIN=1/VTIME=0. Raw mode is active only during editing — restored before running child processes.
- **History** (`history.go`): Persists to `~/.gosh_history`. Skips consecutive duplicates and blank lines. Capped at 1000 entries. File created with mode 0600.
- **Editor** (`editor.go`): Emacs-style key bindings — Ctrl-A/E (Home/End), Ctrl-B/F (Left/Right), Ctrl-K/U (kill to EOL/BOL), Ctrl-W (kill word), Ctrl-L (clear screen), Ctrl-C (cancel), Ctrl-D (EOF on empty / delete char). Up/Down arrows navigate history. Escape sequences decoded for arrow keys, Home/End, Delete.
- **Tab completion** (`editor.go` + `complete.go`): Editor accepts a `CompleteFunc` callback from the shell. Command-position words (first word or after `|`, `;`, `&`) complete from builtins + user-defined functions + PATH executables. Argument-position words complete filenames (directories get `/` suffix). Single match inserts directly with trailing space or `/`. Multiple matches insert the longest common prefix; double-tab displays all candidates in columns (using `TIOCGWINSZ` for terminal width).
- Non-interactive mode (piped input) falls back to `bufio.Scanner` with no editing.
